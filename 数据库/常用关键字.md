# 一、MySQL 新手高频关键字速查表（分类整理）


## 1、建表 / 改表 / 删表类（DDL 数据定义语言）

| 关键字 | 翻译 | 作用 | 示例 & 注意事项 |
|--------|------|------|-----------------|
| `CREATE TABLE` | 创建表 | 新建一张数据表 | `CREATE TABLE tb_college (...)` |
| `IF NOT EXISTS` | 如果不存在 | 建表前判断，表已存在不报错 | `CREATE TABLE IF NOT EXISTS tb_college` |
| `ALTER TABLE` | 修改表 | 表建好后，改字段、加约束、加索引 | `ALTER TABLE tb_student ADD ...` |
| `ADD` | 新增 | 新增字段 / 新增约束 | `ALTER TABLE tb ADD col_id INT;` / `ADD CONSTRAINT ...` |
| `MODIFY` | 修改字段属性 | 修改字段类型、长度、属性，不改字段名 | `ALTER TABLE tb MODIFY stu_name VARCHAR(30);` |
| `CHANGE` | 修改字段名+类型 | 同时修改字段名称和类型 | `ALTER TABLE tb CHANGE old_name new_name VARCHAR(20);` |
| `DROP` | 删除 | 删除字段 / 删除约束 / 删除表 | `DROP COLUMN addr` / `DROP FOREIGN KEY fk_name` / `DROP TABLE tb` |
| `DROP TABLE` | 删除整张表 | 彻底删除表结构和所有数据 | `DROP TABLE IF EXISTS tb_college;` |
| `IF EXISTS` | 如果存在 | 删除前判断，表不存在不报错 | `DROP TABLE IF EXISTS tb_college;` |
| `ENGINE` | 存储引擎 | 指定表的存储机制 | `ENGINE=InnoDB`；InnoDB 支持事务和外键，开发必用 |
| `CHARSET` / `CHARACTER SET` | 字符集 | 指定文字编码 | `DEFAULT CHARSET=utf8mb4`；统一用 utf8mb4 支持中文和表情 |

## 2、字段属性修饰类（建表时跟在字段后面）

| 关键字 | 翻译 | 作用 | 示例 & 注意事项 |
|--------|------|------|-----------------|
| `UNSIGNED` | 无符号 | 数字类型只能存 0 和正数，数值上限翻倍 | `INT UNSIGNED`；仅适用于整数类型，字符串/日期不能用 |
| `NOT NULL` | 不能为空 | 插入数据时该字段必须填写，不允许空值 | `stu_name VARCHAR(20) NOT NULL`；主键自带 NOT NULL，不用重复写 |
| `DEFAULT` | 默认值 | 不填该字段时自动填充预设值 | `stu_sex BOOLEAN DEFAULT 1`；字符串用单引号 `DEFAULT ''`；日期用 `DEFAULT CURRENT_TIMESTAMP` |
| `AUTO_INCREMENT` | 自增 | 整数主键自动从 1 开始递增，不用手动填编号 | `col_id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY`；一张表只能一个自增字段，且必须是主键 |
| `COMMENT` | 注释 | 给字段/表加中文说明，方便理解业务含义 | `COMMENT '学院名称'`；内容用单引号包裹 |

## 3、约束类（表结构规则，保证数据合法性）

| 关键字 | 翻译 | 作用 | 示例 & 注意事项 |
|--------|------|------|-----------------|
| `PRIMARY KEY` | 主键约束 | 唯一标识一行数据，不能重复、不能为空 | `PRIMARY KEY (stu_id)`；一张表只能一个主键，自动创建索引 |
| `CONSTRAINT` | 约束（命名） | 给外键/唯一约束起自定义名字，方便后期删除排错 | `CONSTRAINT fk_stu_col`；约束名全局不能重复，习惯带 `fk_` / `uk_` 前缀 |
| `FOREIGN KEY` | 外键约束 | 本表字段关联另一张表主键，保证数据合法 | `FOREIGN KEY (col_id)`；必须是 InnoDB 引擎，关联字段类型必须完全一致 |
| `REFERENCES` | 引用/关联 | 外键配套，指定关联的目标表和主键 | `REFERENCES tb_college(col_id)`；格式：`目标表(目标主键)` |
| `UNIQUE` | 唯一约束 | 字段值不能重复，但允许一个 NULL | `stu_phone VARCHAR(11) UNIQUE`；一张表可以多个唯一约束 |
| `CHECK` | 检查约束 | 限制字段取值范围，不符合规则报错 | `CHECK(stu_sex IN(0,1))`；MySQL 5.7 忽略，8.0 才完全生效 |

## 4、数据增删改查类（DML 数据操作语言）

| 关键字 | 翻译 | 作用 | 示例 & 注意事项 |
|--------|------|------|-----------------|
| `SELECT` | 查询 | 读取表中数据 | `SELECT stu_id, stu_name FROM tb_student;` |
| `FROM` | 来自 | 指定查询/操作的表 | `FROM tb_student` |
| `WHERE` | 条件 | 筛选符合条件的数据行 | `WHERE col_id = 1`；UPDATE/DELETE 不加 WHERE 会操作全表，**极度危险** |
| `INSERT INTO` | 插入 | 新增一行/多行数据 | `INSERT INTO tb_college(col_name) VALUES('计算机学院');` |
| `VALUES` | 值 | 配合 INSERT 写入具体数据 | `VALUES(1, '张三')` |
| `UPDATE` | 更新 | 修改已有数据 | `UPDATE tb_student SET stu_name='李四' WHERE stu_id=1;` |
| `SET` | 设置 | 配合 UPDATE 指定修改的字段和值 | `SET stu_name='李四'` |
| `DELETE` | 删除 | 删除表中的行数据 | `DELETE FROM tb_student WHERE stu_id=1;`；不加 WHERE 清空整张表 |
| `AND` | 并且 | 多个条件同时满足 | `WHERE col_id=1 AND stu_sex=1` |
| `OR` | 或者 | 任意一个条件满足即可 | `WHERE col_id=1 OR col_id=2` |
| `LIKE` | 模糊匹配 | 字符串模糊查询 | `WHERE stu_name LIKE '张%'`；`%` 代表任意字符 |
| `IN` | 在集合内 | 匹配多个值 | `WHERE col_id IN(1, 2, 3)` |
| `ORDER BY` | 排序 | 按指定字段排序 | `ORDER BY stu_id ASC`（升序）/ `DESC`（降序） |
| `GROUP BY` | 分组 | 按字段分组，配合聚合函数统计 | `GROUP BY col_id` |
| `HAVING` | 分组后筛选 | 对 `GROUP BY` 分组统计后的结果做过滤，**可以使用聚合函数** | 只能跟在 GROUP BY 后面；和 WHERE 区分：WHERE 过滤原始数据，HAVING 过滤分组后的统计结果 `having 平均数 > 90` 这里平均数 不能加引号 |
| `LIMIT` | 限制行数 | 限制返回行数，用于分页 | `LIMIT 0, 10` 取前 10 条 |
| `DISTINCT` | 去重 | 查询结果去除重复行 | `SELECT DISTINCT col_id FROM tb_student` |
| `AS` | 别名 | 给字段/表起别名 | `SELECT stu_name AS 姓名 FROM tb_student` |
| `UNION`     | 联合查询     | 合并**多条 SELECT 查询结果**，自动去除重复数据 | 两条查询列数、字段类型必须完全一致；会自动去重，性能略低 |
| `UNION ALL` | 全量联合查询 | 合并多条 SELECT，**保留全部重复数据**          | 无去重逻辑，速度比 UNION 快，日常优先使用                |

## 5、多表关联类（JOIN 查询）

| 关键字 | 翻译 | 作用 | 示例 & 注意事项 |
|--------|------|------|-----------------|
| `INNER JOIN` / `JOIN` | 内连接 | 只返回两张表都匹配的数据 | `tb_student JOIN tb_college ON ...` |
| `LEFT JOIN` | 左连接 | 左表全部保留，右表匹配不上显示 NULL | `tb_student LEFT JOIN tb_college ON ...` |
| `RIGHT JOIN` | 右连接 | 右表全部保留，左表匹配不上显示 NULL | 实际开发用得少，一般用 LEFT JOIN 替代 |
| `ON` | 连接条件 | JOIN 专用，指定两张表的关联条件 | `ON tb_student.col_id = tb_college.col_id` |
| `GROUP BY` | 分组 | 按字段分组，配合聚合函数统计 | `GROUP BY col_id` |
| `ROLLUP` | 分组汇总 | 配合 `GROUP BY`，自动生成**小计、合计汇总行** | 仅 InnoDB 支持；写在 GROUP BY 末尾；会在结果最后增加总合计行；多字段分组会逐级生成分组小计 用法 ` GROUP BY col_id WITH ROLLUP` |

## 6、聚合统计函数类

| 关键字/函数 | 翻译 | 作用 | 示例 |
|-------------|------|------|------|
| `COUNT()` | 计数 | 统计行数 | `SELECT COUNT(*) FROM tb_student;` |
| `SUM()` | 求和 | 数值字段求和 | `SELECT SUM(score) FROM tb_score;` |
| `AVG()` | 平均值 | 数值字段求平均值 | `SELECT AVG(score) FROM tb_score;` |
| `MAX()` | 最大值 | 取字段最大值 | `SELECT MAX(score) FROM tb_score;` |
| `MIN()` | 最小值 | 取字段最小值 | `SELECT MIN(score) FROM tb_score;` |
| `ROUND()`     | 四舍五入取整 | 对小数四舍五入，可指定保留小数位数 | `ROUND(25.3)=25`、`ROUND(25.6)=26`；`ROUND(12.345,2)` 保留 2 位小数 = 12.35；不适合计算周岁，会虚增年龄 |
| `TRIM()`      | 去除首尾空格 | 清除字符串开头、结尾的空白空格；也可指定删除指定字符 | `TRIM(stu_name)`；仅去除首尾，中间空格不动；MySQL 还支持 LTRIM（只去左边）、RTRIM（只去右边） |
| `CURDATE()`     | 获取当前日期 | 返回系统今日日期（仅年月日，无时分秒）               | `CURDATE()`，只输出 `2026-06-24`                 |
| `DATEDIFF(a,b)` | 日期差值计算 | 计算两个日期相差天数，格式 `DATEDIFF(大日期,小日期)` | `DATEDIFF(CURDATE(), stu_birth)`，得到活了多少天 |
| `FLOOR()`       | 向下取整     | 舍弃小数，只保留整数部分，直接截断小数               | `FLOOR(25.9) = 25`；算年龄专用，不会虚岁 + 1     |
| `COALESCE()`  | 空值替换函数 | 接收多个参数，返回**第一个不为 NULL**的值；专门用来处理 NULL 空值 | `COALESCE(score, 0)`；参数可以写多个；仅识别 NULL，空字符串`''`不算 NULL |

完整组合解释：`FLOOR(DATEDIFF(CURDATE(), stu_birth) / 365) AS 年龄`

# 新手记忆优先级建议

1. **第一优先级（建表必背）**：`NOT NULL`、`DEFAULT`、`PRIMARY KEY`、`FOREIGN KEY`、`REFERENCES`、`CONSTRAINT`、`UNSIGNED`、`AUTO_INCREMENT`、`COMMENT`
2. **第二优先级（改表必背）**：`ALTER TABLE`、`ADD`、`MODIFY`、`DROP`、`CHANGE`
3. **第三优先级（查询必背）**：`SELECT`、`FROM`、`WHERE`、`AND`/`OR`、`LIKE`、`IN`、`ORDER BY`、`LIMIT`、`JOIN`、`ON`
4. **第四优先级（数据操作）**：`INSERT INTO`、`VALUES`、`UPDATE`、`SET`、`DELETE`



# 二、MySQL 查询语句完整固定执行顺序（必背）

## 1、完整标准书写顺序（一条 SELECT 完整结构，不能乱序）

```
SELECT  [DISTINCT] 字段/表达式/聚合函数 AS 别名
FROM 表名 [AS 表别名]
[JOIN 关联表 ON 关联条件]
[WHERE 原始数据行过滤条件]
[GROUP BY 分组字段]
[WITH ROLLUP]
[HAVING 分组后聚合结果过滤]
[ORDER BY 排序字段 ASC/DESC]
[LIMIT 偏移量,条数]
```

## 2、从上到下每一段作用、规则、易错点表格

| 书写顺序 | 关键字模块  | 核心作用                               | 关键限制 & 易错点                                            |
| :------- | :---------- | :------------------------------------- | :----------------------------------------------------------- |
| 1        | SELECT      | 指定要查询展示的列、计算表达式、起别名 | 1. 可写`*`查全部字段2. 聚合函数`AVG/COUNT/FLOOR/ROUND`写这里3. 别名仅在`HAVING/ORDER BY`可用，`WHERE`不能用 |
| 2        | FROM        | 指定查询哪一张 / 多张表                | 必须在 WHERE 前面；可给表起别名简化书写                      |
| 3        | JOIN ... ON | 多表关联查询                           | 关联条件写在`ON`，不要写进 WHERE；分 INNER/LEFT/RIGHT JOIN   |
| 4        | WHERE       | **分组前过滤原始数据**                 | 1. 不能使用聚合函数（AVG/COUNT 等）2. 不能使用 SELECT 里的字段别名3. 用于过滤单行原始数据（课程号、生日、性别等） |
| 5        | GROUP BY    | 按字段分组，配合聚合函数统计           | 1. SELECT 中非聚合字段必须出现在 GROUP BY 后2. 不写 GROUP BY 不能用 AVG/COUNT/SUM |
| 6        | WITH ROLLUP | 分组后自动生成小计、总计行             | 紧跟 GROUP BY 末尾；会产生 NULL 合计行，搭配 IFNULL 美化     |
| 7        | HAVING      | **分组后过滤聚合统计结果**             | 1. 专门筛选 COUNT、AVG 等计算出来的数值2. 可以直接使用 SELECT 定义的字段别名（不加引号）3. 必须存在 GROUP BY 才能使用 HAVING |
| 8        | ORDER BY    | 对最终查询结果排序                     | 1. ASC 升序（默认），DESC 降序2. 支持使用 SELECT 字段别名    |
| 9        | LIMIT       | 限制返回行数，分页                     | 格式 `LIMIT 起始下标,展示条数`；写在整条 SQL 最后            |

## 3、数据库底层真实执行顺序（和书写顺序不一样，理解为什么有各种限制）

底层执行流程：

```
FROM → JOIN → WHERE → GROUP BY → ROLLUP → 聚合计算(AVG/COUNT) → HAVING → SELECT（生成别名）→ ORDER BY → LIMIT
```

### 对应限制解释（新手最难理解的点）

1. WHERE 不能用聚合函数

   WHERE 在 GROUP BY、聚合计算之前执行，此时平均分、总数还没算出来，识别不了 AVG ()。

2. WHERE 不能用 SELECT 别名

   SELECT 是倒数第三步才执行，WHERE 执行时别名还不存在。

3. HAVING 可以用别名

   HAVING 在聚合计算后、SELECT 之前，MySQL 提前识别字段别名，允许直接使用。

4. ORDER BY 可以用别名

   

   ORDER BY 在 SELECT 之后执行，别名已经生成，完全支持。

## 4、完整标准示例（覆盖全部关键字，对照顺序）

```
-- 完整全覆盖示例
SELECT DISTINCT
    tr.stu_id AS 学号,
    ROUND(AVG(tr.score),2) AS 平均分
FROM tb_record tr
LEFT JOIN tb_student ts ON tr.stu_id = ts.stu_id
WHERE tr.cou_id IN (1111,2222,3333) -- 原始数据过滤
GROUP BY tr.stu_id WITH ROLLUP       -- 分组+合计行
HAVING 平均分 >= 90                  -- 分组统计结果过滤
ORDER BY 平均分 ASC                  -- 排序
LIMIT 0,10;                          -- 只取前10条
```

## 5、DDL 建表 / 改表关键字固定顺序（补充）

### （1）. CREATE TABLE 顺序

```
CREATE TABLE [IF NOT EXISTS] 表名(
    字段1 类型 修饰符(NOT NULL/UNSIGNED/DEFAULT/AUTO_INCREMENT) COMMENT '',
    字段2 ... ,
    主键约束 PRIMARY KEY(字段),
    外键约束 CONSTRAINT 约束名 FOREIGN KEY(本表字段) REFERENCES 关联表(主键)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### （2）. ALTER TABLE 修改表顺序

```
ALTER TABLE 表名
ADD 字段/ADD CONSTRAINT 约束
MODIFY 字段 新类型属性
CHANGE 旧字段名 新字段名 类型
DROP COLUMN 字段 / DROP FOREIGN KEY 约束名;
```

## 6、DML 增删改固定顺序

1. 新增：`INSERT INTO 表(字段1,字段2) VALUES(值1,值2);`
2. 修改：`UPDATE 表 SET 字段=值 WHERE 条件;`
3. 删除：`DELETE FROM 表 WHERE 条件;`

## 7、新手必记 3 条核心规则

1. 筛选原始数据用`WHERE`，筛选统计后的数字用`HAVING`；
2. 聚合函数`AVG/COUNT`必须搭配`GROUP BY`，且只能写在 SELECT/HAVING；
3. 整条查询 SQL，`LIMIT`永远写在最后，不能放在任何关键字前面。

# 四、MySQL 常用函数完整表格



## 1、常用字符串函数

| 函数                            | 翻译                | 作用                                         | 示例 & 注意事项                                              |
| :------------------------------ | :------------------ | :------------------------------------------- | :----------------------------------------------------------- |
| `CONCAT()`                      | 字符串拼接          | 拼接多个字符串为一整个字符串                 | `CONCAT(stu_name,'(',stu_id,')')`；任意参数为 NULL 整体返回 NULL |
| `FORMAT()`                      | 数值格式化          | 数字转为带千分位字符串，指定保留小数位数     | `FORMAT(score,2)` 保留 2 位小数                              |
| `TO_BASE64()` / `FROM_BASE64()` | Base64 编码 / 解码  | 字符串 Base64 加密、解密                     | 用于简单文本加密传输                                         |
| `BIN()` / `OCT()` / `HEX()`     | 进制转字符串        | 数字转二进制 / 八进制 / 十六进制文本         | `HEX(10)` 返回 `A`                                           |
| `LOCATE()`                      | 查找子串位置        | 返回子串首次出现下标，找不到返回 0           | `LOCATE('张',stu_name)`                                      |
| `LEFT()` / `RIGHT()`            | 截取左右字符        | 截取字符串左侧 / 右侧指定长度字符            | `LEFT(stu_name,1)` 取姓氏                                    |
| `LENGTH()` / `CHAR_LENGTH()`    | 字节长度 / 字符长度 | LENGTH 统计字节；CHAR_LENGTH 统计字符        | 中文 UTF-8 一个汉字占 3 字节                                 |
| `LOWER()` / `UPPER()`           | 转小写 / 转大写     | 字母全部转为小写、大写                       | `LOWER(cou_name)`                                            |
| `LPAD()` / `RPAD()`             | 左右填充字符        | 长度不足时，左侧 / 右侧填充指定字符补齐长度  | `LPAD(stu_id,6,'0')` 学号补零                                |
| `LTRIM()` / `RTRIM()`           | 清除左空格 / 右空格 | 只删除字符串开头 / 末尾空白                  | 搭配 TRIM 一次性清除首尾空格                                 |
| `ORD()` / `CHAR()`              | 字符编码互转        | ORD 取字符 ASCII 码；CHAR 根据编码返回字符   | `ORD('A')`、`CHAR(65)`                                       |
| `STRCMP()`                      | 字符串比较          | 两串相等返回 0，前者小返回 - 1，前者大返回 1 | `STRCMP('a','b')` 结果 - 1                                   |
| `SUBSTRING()`                   | 截取子串            | 从指定下标截取一段字符串                     | `SUBSTRING(stu_name,2)` 去掉第一个字                         |

## 2、常用数值函数

| 函数                                                         | 翻译                | 作用                                        | 示例 & 注意事项                        |
| :----------------------------------------------------------- | :------------------ | :------------------------------------------ | :------------------------------------- |
| `ABS()`                                                      | 绝对值              | 返回数字非负绝对值                          | `ABS(score-60)` 计算差值               |
| `CEILING()` / `FLOOR()`                                      | 向上取整 / 向下取整 | CEIL 小数直接进 1；FLOOR 直接舍弃小数       | 计算周岁只用 FLOOR                     |
| `CONV()`                                                     | 进制转换            | 将数字从一种进制转为另一种进制              | `CONV('1010',2,10)` 二进制转十进制     |
| `CRC32()`                                                    | 循环冗余校验        | 生成数字格式 CRC 校验值，用于校验数据完整性 | 多用于文件、字符串校验                 |
| `EXP()` / `LOG()` / `LOG2()` / `LOG10()`                     | 指数、对数运算      | 自然指数、自然对数、以 2/10 为底对数        | 数学计算场景使用                       |
| `POW()`                                                      | 幂运算              | 计算底数的 N 次方                           | `POW(2,3)` = 8                         |
| `RAND()`                                                     | 随机数              | 返回 0 ≤ x < 1 的随机浮点数                 | `FLOOR(RAND()*100)` 生成 0-99 随机整数 |
| `ROUND()`                                                    | 四舍五入            | 小数四舍五入，可指定保留小数位数            | `ROUND(95.67,1)` 保留 1 位小数         |
| `SQRT()`                                                     | 平方根              | 计算数字算术平方根                          | `SQRT(100)` = 10                       |
| `TRUNCATE()`                                                 | 截断小数            | 直接砍掉指定位数后的小数，不四舍五入        | `TRUNCATE(99.999,1)` → 99.9            |
| `SIN()` / `COS()` / `TAN()` / `COT()` / `ASIN()` / `ACOS()` / `ATAN()` | 三角函数            | 正弦、余弦、正切、余切及反三角函数          | 数学几何计算专用                       |

## 3、常用时间日期函数

| 函数                                            | 翻译                               | 作用                                                         | 示例 & 注意事项                              |
| :---------------------------------------------- | :--------------------------------- | :----------------------------------------------------------- | :------------------------------------------- |
| `CURDATE()` / `CURTIME()` / `NOW()`             | 当前日期 / 当前时间 / 当前完整时间 | CURDATE 仅年月日；CURTIME 仅时分秒；NOW 包含日期 + 时间      | `CURDATE()` 格式`2026-06-25`                 |
| `ADDDATE()` / `SUBDATE()`                       | 日期加 / 日期减                    | 对日期增加、减少天数 / 年月                                  | `ADDDATE(CURDATE(),30)` 30 天后日期          |
| `DATE()` / `TIME()`                             | 提取日期 / 提取时间                | 从完整时间字符串中单独拆分出日期、时分秒                     | `DATE(NOW())` 只保留年月日                   |
| `YEAR()` / `MONTH()` / `DAY()`                  | 提取年 / 月 / 日                   | 单独取出日期中的年份、月份、几号                             | `YEAR(stu_birth)`                            |
| `HOUR()` / `MINUTE()` / `SECOND()`              | 提取时 / 分 / 秒                   | 单独取出时间的时、分、秒数值                                 | `HOUR(CURTIME())`                            |
| `DATEDIFF()` / `TIMEDIFF()` / `TIMESTAMPDIFF()` | 时间差值计算                       | DATEDIFF 算相差天数；TIMEDIFF 算时分秒差；TIMESTAMPDIFF 可指定年 / 月 / 日单位 | `DATEDIFF(CURDATE(),stu_birth)` 计算存活天数 |
| `MAKEDATE()` / `MAKETIME()`                     | 构造日期 / 构造时间                | 根据年份 + 天数生成日期；根据时分秒生成时间                  | `MAKEDATE(2026,100)`                         |

## 4、常用流程控制函数

| 函数       | 翻译          | 作用                                    | 示例 & 注意事项                        |
| :--------- | :------------ | :-------------------------------------- | :------------------------------------- |
| `IF()`     | 三元判断函数  | 条件成立返回值 1，不成立返回值 2        | `IF(stu_sex, '男','女')`               |
| `IFNULL()` | NULL 替换函数 | 字段为 NULL 则返回替代值，否则返回原值  | `IFNULL(score,0)` 空成绩显示 0         |
| `NULLIF()` | 等值置空函数  | 两表达式相等返回 NULL，否则返回第一个值 | `NULLIF(score,0)` 分数为 0 时返回 NULL |

## 5、其他常用函数

| 函数                           | 翻译                  | 作用                                                      | 示例 & 注意事项                  |
| :----------------------------- | :-------------------- | :-------------------------------------------------------- | :------------------------------- |
| `MD5()` / `SHA1()` / `SHA2()`  | 哈希加密函数          | 生成字符串不可逆哈希摘要，用于密码存储                    | 密码不直接明文存储，使用哈希加密 |
| `CHARSET()` / `COLLATION()`    | 字符集 / 校对规则查询 | 查询字段、字符串使用的字符集与排序规则                    | `CHARSET(stu_name)`              |
| `USER()` / `CURRENT_USER()`    | 当前登录用户          | 获取当前数据库登录账号                                    | 排查权限问题使用                 |
| `DATABASE()`                   | 当前数据库名          | 返回正在操作的数据库名称                                  | 多库切换时查看当前库             |
| `VERSION()`                    | 数据库版本            | 获取 MySQL 服务版本号                                     | 区分语法兼容特性                 |
| `FOUND_ROWS()` / `ROW_COUNT()` | 查询行数 / 影响行数   | FOUND_ROWS 获取匹配数据总量；ROW_COUNT 获取增删改影响行数 | 分页、增删改后统计数量           |
| `LAST_INSERT_ID()`             | 最后自增 ID           | 获取上一条 INSERT 语句生成的自增主键值                    | 新增数据后快速拿到主键           |
| `UUID()` / `UUID_SHORT()`      | 唯一 ID 生成          | 生成全局不重复 UUID 字符串 / 短数字 ID                    | 无自增主键时做唯一标识           |
